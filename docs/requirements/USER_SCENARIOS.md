## 1. 시나리오 (5H1W 형식)

### 시나리오 S-01: 영어 화상회의 중 실시간 자막 따라가기

> **(Who)** 영어 회의에 참여한 한국어 모국어 화자가
> **(Why)** 빠르게 진행되는 영어 발화의 내용을 놓치지 않고 의사결정에 참여하기 위하여
> **(What)** 마이크(또는 시스템 오디오)로 들어오는 다중 화자의 영어 발화를 화자별로 색·라벨이 구분된 한국어 자막으로 실시간 확인하는 것을 원한다.

**상세 흐름**:
1. Zoom/Google Meet 회의가 시작되기 직전, RVT 앱을 실행한다.
2. 마이크(또는 BlackHole 같은 가상 오디오) 입력 소스를 선택하고 "시작" 버튼을 누른다.
3. 회의가 진행됨에 따라 다른 참여자(2~4명)가 번갈아 말한다.
4. RVT는 각 발화를 자동으로 화자별로 분리하고, 한국어 자막을 **화자A: ... / 화자B: ...** 형태로 실시간 표시한다.
5. 본인은 자막 창을 곁눈질하며 발언 내용을 따라간다.
6. 본인이 발언할 차례가 오면 자막 보조를 받아 응답한다.

**암묵적 기대 (반드시 만족되어야 할 것)**:
- 자막 지연이 2~3초를 넘으면 회의 흐름을 못 따라간다 → **NFR-001 (지연)**
- 번역이 핵심 의미를 왜곡하면 잘못된 의사결정 위험 → **NFR-002 (정확도)**
- 화자가 자주 바뀌어도 라벨이 안정되어야 함 → **FR-005 (화자 일관성)**

---

### 시나리오 S-02: 회의 종료 후 회의록 다시 보기

> **(Who)** 회의를 마친 사용자가
> **(Why)** 회의에서 결정된 사항·액션 아이템을 확인하고 동료에게 공유하기 위하여
> **(What)** 세션 종료 시 자동으로 저장된 화자별 대화 로그를 텍스트 파일로 열어 검토하는 것을 원한다.

**상세 흐름**:
1. 회의가 끝나면 RVT의 "종료" 버튼을 누른다.
2. RVT는 자동으로 세션 로그를 텍스트 파일로 저장한다 (예: `~/Documents/RVT/2026-05-25_meeting.txt`).
3. 파일 내용은 시간 + 화자 + 영어 원문 + 한국어 번역 형태로 정렬돼 있다.
4. 사용자는 텍스트 에디터로 파일을 열어 액션 아이템을 찾는다.
5. 필요한 부분을 복사해 Notion/Slack에 공유한다.

**암묵적 기대**:
- 파일 위치를 매번 묻지 않고 기본 폴더에 저장 → **FR-006 (자동 저장)**
- 화자별 라벨이 파일에도 보존 → **FR-007 (로그 포맷)**

---

### 시나리오 S-03: 회의 시작 전 마이크/오디오 설정

> **(Who)** RVT를 처음 사용하는 시점의 사용자가
> **(Why)** 회의가 시작되기 전에 입력 소스가 올바른지 확인하기 위하여
> **(What)** 마이크 입력 레벨이 실시간으로 표시되고 입력 소스를 드롭다운으로 바꿀 수 있기를 원한다.

**상세 흐름**:
1. 앱 실행 시 메인 화면에 "오디오 입력" 영역이 보인다.
2. 드롭다운에서 사용 가능한 입력 디바이스 목록이 표시된다 (마이크, BlackHole 등).
3. 선택한 입력의 음성 레벨이 미터(meter) 형태로 시각화된다.
4. 사용자는 본인이 말해보거나 시스템 오디오를 틀어 입력이 정상 작동함을 확인한다.
5. 확인 후 "시작" 버튼이 활성화된다.

**암묵적 기대**:
- 입력이 없으면 "시작" 버튼이 비활성화 → **FR-008 (입력 검증)**
- macOS 마이크 권한 요청이 자연스럽게 → **NFR-006 (플랫폼 통합)**

---

### 시나리오 S-04 (예외/실패 케이스): 네트워크/모델 오류

> **(Who)** 회의 중인 사용자가
> **(Why)** 일시적 네트워크 단절이나 STT 모델 오류 상황에서도 회의 흐름을 잃지 않기 위하여
> **(What)** 오류 발생 시 자막이 멈추지 않고, 명확한 오류 메시지가 표시되며, 가능한 한 자동 복구되기를 원한다.

**상세 흐름 (예외 경로)**:
1. 회의 중 인터넷 연결이 불안정해진다.
2. 번역 API 호출이 실패한다.
3. RVT는 영어 원문 STT는 계속 표시하되, 번역 부분에는 "⚠ 번역 일시 중단 — 재시도 중" 표시.
4. 연결이 복구되면 자동으로 밀린 발화부터 번역을 재개한다.
5. 사용자는 회의를 중단할 필요가 없다.

**암묵적 기대**:
- 부분 실패가 전체 중단으로 이어지지 않음 → **NFR-005 (신뢰성)**
- 오류는 침묵하지 않고 사용자에게 통지 → **FR-009 (오류 통지)**

## 2. 시나리오 우선순위

| 시나리오 | 우선순위 | 근거 |
|---|---|---|
| S-01 (실시간 자막) | **필수 (MVP)** | RVT 존재 이유 자체 |
| S-02 (로그 저장) | **필수 (MVP)** | 회의 후 활용 가치의 핵심 |
| S-03 (오디오 설정) | 중요 | S-01 실행 전제 조건 |
| S-04 (오류 복구) | 권장 | 안정성 확보, 시간 부족 시 단순 오류 표시로 축소 가능 |
